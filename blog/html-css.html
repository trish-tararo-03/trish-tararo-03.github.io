CSS Positions

The differences between relative, aboslute and fixed positioning.

CSS is a beautiful and powerful tool we use to add flair and style to our projects. One of the properties we use is the position property. This allows us to manipulate the position of any element using the top, right, bottom, left and z-index values. The five values that the position property can have are:

  • Static
  • Relative
  • Absolute
  • Fixed
  • Sticky

By default, all HTML elements will have a position: static property. This simply means that the elements will follow the normal flow of your HTML document. We can then use the position property to manipulate the final position of an element on the page. In this article we will look at 3 different values: Relative, Absolute and Fixed. To explain their differences, we must explain their individual uses.

Relative positioning

Absolute Positioning

Now imagine you have 5 boxes like the example below: The dotted line represents the big box that contains the other boxes.
With position:absolute we can place the green box whereever we want! That is absolute!
However, there are a couple of things to note:
One is that the green box needs a parent, (or a big box) to reference it position to.
Two is that it would be ignored by the other boxes.

Fixed Positioning

Now take a look at the example below:

Imagine you have two boxes, a big box, and the green one within it. With position:fixed we can use a hot glue gun to stick the botton of the green box to the big box. This way, it's position wont change, even if you throw the big box!

The Difference?

  • Using relative positioning we can move a box relative to its original position.
  • Using absolute positioning, we can move a box anywhere we want, as long as it has a parent box for reference, and it is okay with being ignored by the other boxes.
  • Using fixed positioning, we can glue the box down so that it wont move, no matter what happens.
